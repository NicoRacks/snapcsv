SnapCSV

Rules

  • Sentence case always. Never title case on buttons.
  • Never exceed 2 font weights per view.
  • Monospace (JetBrains Mono) only for: license key input, exported filename.
  • Sans (IBM Plex Sans) and Mono (JetBrains Mono) bundled as variable .ttf files in assets/fonts/ — zero external imports, no CDN, no CSP changes.

09

Do's and don'ts

Quick scan

Do

  • CSS variables only — never hardcode hex outside :root
  • Sans = IBM Plex Sans, Mono = JetBrains Mono — system stack as fallback
  • Green #1adb72 exclusively for upgrade / limit
  • Green #22c55e exclusively for export success
  • Monospace only for license keys + filenames
  • type="button" on every <button>
  • Sentence case on all button labels

Don't

  • No box-shadow — ever
  • No border-radius > 8px
  • No transitions > 150ms
  • No font families besides IBM Plex Sans + JetBrains Mono (and the system fallback)
  • No purple, amber, or orange
  • No title case on button labels
  • No popup width changes — 360px is fixed
  • No showing footer nudge AND upgrade panel together
